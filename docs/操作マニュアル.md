# ソッコーサイト 操作マニュアル

**バージョン 1.2（2026-07-19）｜対象: オペレーター・承認者（付録: 管理者向け）**

---

## 1. ソッコーサイトとは

ヒアリング内容を入力するだけで、AIが事務所・店舗のホームページを自動生成し、品質チェック・承認を経てワンクリックで公開できる社内ツールです。公開後は、店主がLINEでお知らせを更新でき、オペレーターは管理画面から文言修正やお知らせ代理投稿ができます。多業種対応を前提に設計されており、業種テンプレートは順次追加されます。

- 制作ツール（studio）: **https://sokko-studio.vercel.app**
- 業種: **選択不要（AIが自動判定）**。士業・ITは専用の言葉づかい、その他の業種も汎用デザインで対応 ※医療など広告規制の厳しい業種は対応準備中
- かかる時間の目安: 入力30分 → AI生成5分 → 確認・承認 → 公開2分

---

## 2. ログインとアカウント

1. https://sokko-studio.vercel.app を開く
2. メールアドレスとパスワードでログイン（アカウントは管理者が発行します）

アカウントには2種類の役割があります:

| 役割 | できること |
|---|---|
| **オペレーター** | 案件作成・入力・生成・写真・文言編集・お知らせ管理・公開操作 |
| **承認者** | 上記すべて ＋ **公開前の承認・差し戻し**（Step6） |

## 3. サイト制作の流れ（8ステップ）

案件一覧の「＋新しいサイトを作る」から始めます。各ステップは自動保存されるので、途中でブラウザを閉じても続きから再開できます。上部のステッパーで現在地がわかります。

### Step 1: ヒアリング入力

お客様から聞いた内容をA〜Iの項目に入力します。

- **打ち合わせメモの貼り付けで自動入力できます（推奨）**: 画面上部の「打ち合わせメモから自動入力」に議事録やメモをそのまま貼って「AIで下書きを作成」を押すと、各項目にAIが下書きを入れます
  - 入力済みの欄は上書きされません。メモに無い情報（電話番号など）は空欄のまま残ります
  - AIの下書きはあくまで下書きです。**特に★印の項目は必ず確認・調整**してください

- **★印の3項目は特に重要です**（サイトの仕上がりを決めます）:
  - C: お客様が検索しそうな言葉（1行に1つ）
  - F: 参考サイトURL（「こんな雰囲気にしたい」1〜2件）
  - H: 「○○といえば△△」ポジショニング一文
- 未入力のまま「確定して次へ」を押すと、足りない項目が赤字で表示されます
- 途中保存は「下書き保存」

### Step 2: 業種確認・テンプレート・デザイン選択

- **業種の選択は不要です。** Step1の「業務内容の一言説明」からAIが自動判定し、この画面に判定結果（例:「学習塾」）が表示されます。違っていればその場で修正できます
  - 医療機関など広告規制が特に厳しい業種は、Step1確定時に「対応準備中」と案内されます（システム管理者に相談）

業種テンプレート（ページ構成）を確認し、**デザインを3種類から選びます**。メインカラーはStep1の「F. サイトの雰囲気」で選んだ色が使われます。

| デザイン | 雰囲気 | 向いている業種 |
|---|---|---|
| **上質・信頼** | 明朝体の見出し×広い余白×真鍮の装飾ライン | 士業・クリニックなど信頼で選ばれる業種 |
| **先進・近未来** | ダーク背景×発光アクセント×極太タイポ | IT・テック系、新しさを打ち出したい事業 |
| **温かみ・親しみ** | 丸ゴシック×大きな角丸×柔らかな配色 | 飲食・美容・地域密着サービス |

- どれを選んでも**文章・ページ構成・写真は共通**です（見た目だけが変わります）
- 公開後の変更も可能: Step2で選び直し → Step7「変更を再公開する」

### Step 3: コンテンツ自動生成

「コンテンツ生成を開始」を押すと、AIが全ページの文章を書きます（約5分）。

- 生成された文章は、AIが自動で自己チェック（その事業者固有の内容か・検索語が入っているか・業種にふさわしいトーンか・誇大表現がないか）してから保存されます
- **失敗したページがあれば「再実行」**を押すだけでそのページだけやり直せます
- 全ページ完了したら「次へ」

### Step 4: ビジュアル配置（写真）

写真は4つの決まった場所（スロット）にアップロードします:

| スロット | 表示場所 | 推奨 |
|---|---|---|
| ロゴ | ヘッダー | 透過PNG |
| メインビジュアル | トップの背景 | 横長の写真（横1920px以上） |
| 代表者の写真 | 代表・事務所紹介 | **実在の写真のみ**（信頼の要） |
| 事務所・店舗の写真 | アクセス | **実物の写真のみ**（来訪の目印） |

- **写真がなくても公開できます**（写真なしのデザインに自動調整）
- 大きい写真は自動で縮小・軽量化（WebP変換）されるので、そのままアップロードしてOK
- **メインビジュアルは「AIで生成」ボタンでも作れます**: 業種・デザイン・テーマカラーに合う背景画像をAIが生成します（10〜30秒。気に入らなければ何度でも生成し直し・手動写真への差し替え可）
  - ※代表者・事務所の写真は実物のみ。AI生成は使えません（信頼の要のため）

### Step 5: AEO/GEO最適化の確認

AIに見つけてもらうための設定（構造化データ・llms.txt・検索語の織り込みなど）は**すべて自動適用**です。内容を確認して「次へ」を押すだけです。

### Step 6: プレビュー・品質チェック・承認

公開前の最終確認です。3つの順番で進めます:

1. **「プレビューをデプロイ」** — 実際の配信環境にプレビュー版を作ります（数分）。URLが表示されたらお客様への事前確認にも使えます（検索エンジンには載りません）
2. **「チェックを実行」** — 表示速度・アクセシビリティ・リンク切れ・メタ情報・禁止表現などを自動チェック（2〜3分）
   - **要修正（赤）が1件でもあると公開できません**
   - 警告（黄）は承認者の判断で通せます
3. **承認（承認者のみ）** — 「承認して公開へ進む」または「差し戻す」

### Step 7: 公開

「サイトを公開する」ボタン一つで本番公開されます（数分）。

- 公開済みの案件では **「変更を再公開する」** ボタンに変わります — 文言編集・写真差し替え・独自ドメイン設定の反映はここから

### Step 8: 納品

- 公開URL・お客様への納品案内（そのまま送れる文面）・運用メモが表示されます
- **独自ドメイン**の設定欄もここにあります（→ 5章）

## 4. 差し戻しになったら（オペレーター向け）

承認者が差し戻すと、案件は「差し戻し対応中」になり、対象ページと理由が表示されます。

1. **Step 3** を開くと **「差し戻し分を再生成」** ボタンが出ています
2. 押すと、**承認者のコメントをAIが読んで**対象ページだけを書き直します
3. 完了したら「次へ」→ Step 6 で再度「プレビューをデプロイ」→「チェックを実行」→ 承認依頼

入力情報そのものが間違っていた場合は、Step 1 に戻って修正してから再生成してください。

## 5. 公開後の運用

### 5.1 お知らせの更新

**店主（お客様）:** 公式LINEにお知らせしたい内容を送信するだけ。**AIが訪問者向けの文面に清書して返信する**ので、内容を確認して**「OK」と返信すると掲載**されます（約1分でトップと「お知らせ」ページに反映）。

- メモ書きでOK（例:「明日は台風で臨時休業」→ 日付入りの丁寧な文面に清書されます）
- 文面を変えたいときは、修正した内容をもう一度送ると作り直されます／「キャンセル」で取りやめ
- **「OK」と返信するまでサイトには掲載されません**
- テキストのみ対応（画像・スタンプは不可）
- 未登録のLINEから送ると「登録用ID」が返信されます → そのIDを管理者に伝えると利用登録されます

**オペレーター:** 案件一覧の **「お知らせ管理」** から:

- **投稿する** — 代理投稿（約1分で反映）。**「AIで校正」ボタン**でメモ書きを訪問者向けの文面に清書してから投稿できます（結果は投稿前に手直し可能）
- **編集** — 誤字修正など（LINEからの投稿も編集可）
- **非公開にする** — サイトから即座に消えます（間違い投稿の取り消し）

### 5.2 文言のピンポイント修正

案件一覧の **「文言編集」** から、ページごとに見出し・本文・項目のテキストを直接編集できます。

- **変えられるもの:** 文字だけ（ページタイトル・メタ説明・見出し・本文・FAQ・AI向けサイト要約）
- **変えられないもの:** ページ構成・デザイン・写真（壊せない設計）
- 「日本一」「必ず」などの**広告規制上危険な表現は保存できません**（自動ブロック）
- 保存後、**Step 7 の「変更を再公開する」で反映**されます

### 5.3 文章を大きく書き直したいとき

トーンや構成レベルの修正は、承認者に「差し戻し」をしてもらうのが早道です（コメントを書けばAIがその指示どおりに書き直します → 4章）。

### 5.4 写真の差し替え

Step 4 でいつでも差し替え・削除できます。反映は「変更を再公開する」。

### 5.5 独自ドメインの接続

1. **【管理者の作業】** お客様のドメインをCloudflareにゾーン追加（ネームサーバー切替）
2. Step 8 の「独自ドメイン」欄にドメインを入力して保存（例: `suzuki-sr.jp`）
3. Step 7 で「変更を再公開する」→ SSL証明書込みで自動接続。サイト内のURL（canonical・sitemap等）も独自ドメイン基準に切り替わります

## 6. 承認者向けガイド

### 品質チェック結果の読み方

| 表示 | 意味 | 対応 |
|---|---|---|
| ✓ 合格 | 公開基準クリア | 承認できます |
| 警告（黄） | 軽微な注意点（例: 表示速度がやや低め） | 内容を確認のうえ承認可 |
| 要修正（赤） | 公開基準未達（リンク切れ・速度不足・禁止表現など） | 修正されるまで承認ボタンは無効 |

基準値: モバイル表示速度スコア 80未満=要修正・90未満=警告 ／ アクセシビリティ 90未満=要修正

### 承認と差し戻し

- **承認して公開へ進む** — 品質チェック合格時のみ押せます。承認記録が残ります
- **差し戻す** — 対象ページを選び、**理由コメントを必ず記入**（このコメントをAIが読んで書き直すので、具体的に書くほど良くなります。例:「トップの見出しが抽象的。就業規則と助成金の強みが伝わる表現に。実績数字も活かして」）

## 7. よくあるトラブル

| 症状 | 対処 |
|---|---|
| 生成が「失敗」になった | そのページの「再実行」を押す（他のページに影響なし） |
| 生成が「応答なし」のまま | 5分待っても変わらなければ「再実行」 |
| プレビューURLが表示されない | デプロイに数分かかります。ページを再読み込み |
| 公開したのにサイトが変わらない | 反映に1〜2分。ブラウザの再読み込み（スマホは引っ張って更新） |
| お知らせがサイトに出ない | トップの「お知らせ」欄と `/news` ページを確認。管理画面で「非公開」になっていないか確認 |
| LINEで送ったのに掲載されない | AIの清書返信に**「OK」と返信したか**確認（OKするまで掲載されません） |
| 店主のLINEに「登録されていません」と返る | 返信内の「登録用ID」を管理者に伝えて登録を依頼 |
| 保存時に「広告規制上問題になりうる表現」と出る | 指摘された言葉（日本一・必ず 等）を事実ベースの表現に直す |
| ログインできない | パスワードを管理者に確認（リセット可能） |

## 8. 付録: システム管理者向け

### インフラ構成

| 役割 | サービス | 備考 |
|---|---|---|
| 制作ツール本体 | Vercel（`sokko-studio`） | mainへのpushで手動デプロイ運用 |
| データベース/認証/画像 | Supabase（`sokko-site` / gpoxtnyvrjwwtcilubez） | マイグレーションはリポジトリ管理 |
| 生成サイトの配信 | Cloudflare Workers Static Assets | 帯域無料・サイトごとに `site-{slug}` |
| ビルド&デプロイ | GitHub Actions（`hiroshikomori-code/sokko-site`） | studioのボタンから自動起動 |
| LINE連携 | Supabase Edge Function `line-webhook` | シークレットはSupabase secrets |

### 新しいクライアントのLINE設定（現状はMVP: 1チャネル運用）

1. LINE公式アカウント作成 → Messaging API有効化（プロバイダー: caminova）
2. Webhook URL: `https://gpoxtnyvrjwwtcilubez.supabase.co/functions/v1/line-webhook?project={案件ID}`（Webhook利用ON・応答メッセージOFF）
3. チャネルシークレット/アクセストークンを Supabase secrets に設定
4. 店主が友だち追加 → 返信された「登録用ID」を `line_links` に登録

※ 複数クライアントの同時LINE運用（チャネルごとの鍵管理）はPhase 2対応予定。

### アカウント発行

Supabase Auth でユーザー作成 → `users` テーブルに `role`（operator / approver）を登録。

### 運用時の注意

- 生成AIの原価: 1サイトあたり十数万トークン（数十円〜百数十円程度）。`generation_jobs` にトークン数を記録済み（原価集計の土台）
- 品質・公開の全操作は `audit_log` に記録されます
- テンプレート改修後は、既存サイトへは「変更を再公開する」で反映
